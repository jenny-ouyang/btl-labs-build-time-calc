### Data Models

No database. All data is defined as TypeScript constants:

```typescript
// lib/calculator.ts
interface Feature {
  id: string
  label: string
  description: string
  baseHours: number
}

type Complexity = "simple" | "medium" | "complex"

interface CalculationResult {
  selectedFeatures: Feature[]
  complexity: Complexity
  multiplier: number
  totalHours: number
  totalDays: number
  breakdown: { feature: string; hours: number }[]
}
```

### Key Patterns

**Single-page client-side app.** The page.tsx is a Server Component that renders the layout, heading, and footer. The calculator form and results are Client Components (they need useState for form state and results).

**No API routes, no Server Actions.** All calculation logic runs in the browser.

**Component boundaries:**
- `page.tsx` (Server Component) — static layout, metadata, heading, footer
- `_components/calculator-form.tsx` ("use client") — checkboxes, radio buttons, calculate button, manages form state
- `_components/results-display.tsx` ("use client") — results table, progress bar, share button, fade-in animation

**Calculation logic in lib/calculator.ts:**
- Pure functions, no side effects
- Feature list with base hour estimates
- Multiplier lookup by complexity
- Total = sum of selected feature hours * complexity multiplier
- Days = total hours / 6 (productive hours per day)
